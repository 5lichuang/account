# 管理员登录与数据迁移

## 目标范围

第一阶段只新增一个持久化管理员和服务端登录会话，不开放普通用户注册，也不持久化上游账号、API Key、Access Token、AK/SK 或 Cookie。首次启动时没有管理员，浏览器进入初始化页面；管理员创建成功后初始化入口关闭。

## 数据结构

- `auth_users`：保存唯一管理员的规范化用户名、scrypt 密码哈希和创建时间。
- `auth_sessions`：保存随机会话令牌的 SHA-256 摘要、所属用户和固定过期时间，不保存浏览器收到的原始令牌。
- 腾讯云 Docker 使用 `/data/zhangdan.sqlite`，宿主机目录为 `/var/lib/zhangdan`。
- Sites／Worker 部署使用逻辑 D1 绑定 `DB`，执行同一份 Drizzle SQLite 迁移。

## 迁移步骤

1. 发布前备份现有发布包；当前版本没有持久化业务数据，无需迁移上游账号。
2. 安装器创建权限为 `0700` 的 `/var/lib/zhangdan`，并将它挂载到容器 `/data`。
3. 新版本首次收到非健康检查请求时，以 `CREATE TABLE IF NOT EXISTS` 初始化认证表；重复发布不会覆盖现有管理员。
4. 管理员第一次访问 `/setup` 创建账号，成功后自动登录；后续访问 `/setup` 会转到登录页或看板。
5. 发布回滚只切换应用容器，不删除 SQLite 数据卷；旧版本会忽略该数据库。

## 回滚与恢复

- 应用回滚不得删除 `/var/lib/zhangdan/zhangdan.sqlite`。
- 数据库初始化失败时，新容器健康检查仍可用于诊断，但页面和管理 API 必须失败关闭，不能绕过登录。
- 忘记管理员密码的第一版恢复方式是离线备份并重建认证数据库；此操作会删除管理员和全部会话，必须单独确认后由运维执行。

## 验证清单

- 未初始化时首页跳转 `/setup`，且管理 API 返回 `401`。
- 第一个管理员可创建并自动登录；第二次初始化返回冲突，不能创建第二个用户。
- 密码明文和原始会话令牌不出现在数据库、响应体、日志或发布包。
- 错误密码按 IP＋用户名限速；登录错误不区分用户不存在和密码错误。
- 登录 Cookie 包含 `HttpOnly`、`SameSite=Lax`、固定有效期；HTTPS 请求还包含 `Secure`。
- 未登录不能读取、新增、编辑、删除或刷新上游；`/healthz` 继续公开且不返回认证信息。
- 登出后原会话失效；过期会话不能继续访问。
- 容器重启和重新发布后管理员仍存在；上游内存账号仍按当前约束清空。
