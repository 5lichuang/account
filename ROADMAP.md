# 项目路线图

## 当前阶段

多上游余额 MVP 已支持兼容 Bearer 协议、网页访问令牌（Bearer）、阿里云 BSS 与通用网页登录会话（Cookie）余额查询；四类上游均通过构建和端到端回归验证。首次管理员、SQLite 持久化会话和管理接口鉴权已在腾讯云私有容器完成部署与验收；生产仍仅映射 `127.0.0.1:3210`，通过 SSH 隧道访问。

## 当前限制

- 上游配置与凭证只保存在服务进程内存，重启或重新构建后需要重新录入。
- 60 秒刷新由可见浏览器标签页触发，不是服务端 24×7 定时任务；多个标签页会分别轮询。
- 目前只持久化管理员和登录会话；上游配置及凭证仍不持久化，重启后需要重新录入。
- 当前版本仍仅用于可信私有环境；登录不能替代 HTTPS，配置证书和反向代理来源 IP 信任边界前不发布到公网。
- 阿里云真实 AK/SK 联调待运营人员在本地页面录入验证；仓库和项目日志中未保存真实凭证。
- 网页会话类型依赖各上游未公开文档化的站内接口，路径、字段或会话规则可能变化；Cookie 过期后需要人工登录并更新。
- 网页访问令牌第一版不自动登录或续期；Access Token 到期后需要运营人员重新登录上游并手工替换。
- 数据宝真实新 Token 的本地联调待运营人员在页面录入；项目文件、测试和日志中没有保存对话里曾暴露的凭证或身份信息。
- 腾讯云服务器端私有部署和本机 SSH 隧道访问均已验证；访问依赖隧道保持连接，当前不得开放 `3210` 公网端口。
- 腾讯云已运行认证版镜像 `zhangdan:20260719T144629Z`；首次管理员尚未在生产创建，需通过 SSH 隧道访问 `/setup` 完成初始化。
- GitHub 尚未配置 Pull Request 自动检查、开发环境部署或基于 Tag 的生产发布流程。
